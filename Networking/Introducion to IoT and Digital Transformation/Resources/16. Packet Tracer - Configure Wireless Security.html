Packet Tracer - Configure Wireless Security

Objectives

In this activity, you will configure a wireless router for a home network.

Part 1: Modify Basic Security Settings

Part 2: Configure Wireless Security on the Wireless Router

Required Resources

=   Latest version of Cisco Packet Tracer

Background / Scenario

The default setting of a wireless router can easily be found by searching on the internet so the settings should be changed to prevent unauthorized access to your network.

In this activity, you will modify some of the security settings. The configuration of the wireless router in Packet Tracer can provide an example of the interface. The interface varies greatly depending on the model and manufacturer of a wireless router.

Instructions

Part 1: Modify Basic Security Settings

Step 1: Modify the default administrative password.

a.     From the PC, open a web browser from the Desktop and navigate to 192.168.0.1 to access the Home Net wireless router.

b.     Log in to the router using admin as the username and password.

c.     After logging in successfully, click Administration > Management.

d.     Modify the router password to a stronger one. Change the password to aC0mpAny3. Note that the new password has 8 characters with upper and lower-case digits and some of the vowels have been changed to numbers. Click Save Settings at the bottom of that screen.

e.     Log into the router with the new password. Click OK. Click Continue to access the configuration page again.

Step 2: Disable remote management on the wireless router.

For security reasons, remote management, for example access from outside the local network, should be disabled.

a.     Check the current status of remote management on the wireless router. Select Administration > Remote Access. At this time, remote management is enabled.

b.     Select the Hacker PC and navigate to 192.31.7.100 from the web browser. You should be presented with the request for a user name and a password. Enter the credentials of admin and the password aC0mpAny3 and you should be able to access the wireless router configuration page. Close the web browser when done.

c.     Return to the PC and log into the configuration page of the Home Net wireless router as needed to disable Remote Management. (Administration > Management) Select Disabled and click Save Settings. Click Continue.

d.     Return to the Hacker PC and navigate to 192.31.7.100. You should no longer be able to the Home Net wireless router remotely.

Part 2: Configure Wireless Security on the Wireless Router

Step 1: Modify the default SSID name and disable the broadcast feature.

a.     While log into the Home Net wireless router from the PC, click Wireless and modify the SSID name to aCompany.

b.     For SSID Broadcast and click Disabled. Click Save Settings to continue.

Question:

Check the topology. What happened to the wireless connectivity between the Laptop and Home Net?

Answer Area

The Laptop has lost connectivity to Home Net because the Laptop has not been configured with the new SSID name.

Step 2: Configure WPA2 security on the wireless router.

a.     Return to the PC and log into the configuration page of the Home Net wireless router as needed.

b.      Select Wireless > Wireless Security.

c.     Change Security Mode to WPA2 Personal. AES is currently the strongest encryption protocol available. Leave it selected.

d.     Configure the passphrase as aCompWiFi. Click Save Settings.

Step 3: Configure Laptop as a wireless client.

In this step, you will configure Laptop to connect to the wireless network using the security settings you configured on the wireless router.

a.     Select Laptop. Click Desktop > PC Wireless.

b.     Select Profiles > New and add a new profile. Enter a profile name of your choice and click OK.

c.     Click Advanced Setup. Enter the SSID aCompany as Wireless Network Name. Click Next to continue.

d.     Accept the default network settings to use DHCP and click Next.

e.     Modify the Security using the dropdown box to WPA2-Personal and select Next.

f.      Enter the pre-shared key: aCompWiFi and select Next.

g.     Click Save to confirm the new settings.

h.     Click Connect to Network.

i.      If the Laptop does not connect successfully, return to the profile and edit. Check your typing of the SSID name and the pre-shared key.

j.      Close the PC Wireless window when connected.

Step 4: Configure the Wireless Router to support MAC filtering.

a.     From the Desktop, click Command Prompt.

b.     At the prompt, enter ipconfig /all and take note of the IPv4 address and Physical (MAC) addresses for the Wireless0 Connection.

c.     Return to the PC and log into the configuration page of the Home Net wireless router as needed.

d.     Navigate to Wireless > Wireless MAC Filter.

e.     Under the Access Resolution heading, select Enabled and Permit PCs listed below to access wireless network.

f.      Enter the MAC address for the Wireless0 connection on Laptop in the MAC 01: field. Notice the MAC address must be in the XX:XX:XX:XX:XX:XX format. Click Save Settings and click Continue to return to the configuration page.

g.     Reconnect Laptop to the wireless network. (Laptop > click PC Wireless > Profile > select aCompany > click Connect if necessary.)

Step 5: Test the MAC filtering of the Wireless Router

In this step, you will test the MAC filtering by adding a second laptop to the network. You will need change out the fast Ethernet network interface card to a wireless network interface card.

a.     Add a second laptop to the topology. You will need change out the fast Ethernet network interface card to a wireless network interface card.

Note: Do not change the display name of the newly added laptop. The name of the device will affect scoring in Packet Tracer.

b.     Press the power button on Laptop0 to turn it off.

c.     Drag the Ethernet port to the Modules list to remove it.

d.     Drag the WPC300N module to the empty slot on Laptop0 and press the power button to start Laptop0.

Question:

Were you unable to associate with the access point? If not, explain how you would fix it.

Answer Area

Laptop0 is unable to connect to the wireless router because MAC address of the Laptop is the only one specified, Laptop is the only wireless device currently capable of associating with the wireless router. Add the MAC address of Laptop0 to the permit MAC Address filter list.

e.     Configure the new laptop with the security settings in the previous steps to connect Laptop0 to the network.
